# ProlifeLanding

## Mission
Это коммерческий landing page для доставки грузов из Китая в Бишкек.
Главная цель — рост конверсии в лиды и заявки.

## Language
- Всегда отвечай пользователю на русском языке
- Все объяснения, планы, summary и комментарии пиши на русском
- Код и технические идентификаторы оставляй на языке проекта

## Product Priorities
1. Конверсия в лид важнее визуальной избыточности
2. Простота и ясность важнее сложных эффектов
3. Hero, calculator, CTA, trust blocks — зоны наивысшего приоритета
4. Любые изменения должны сохранять быстрый рендер и адаптивность

## Engineering Rules
- Никогда не пушь напрямую в main
- Всегда работай через новую feature-ветку
- Перед коммитом обязательно запускай build
- Перед push показывай список измененных файлов
- Перед PR коротко объясняй что изменилось и зачем
- Не меняй архитектуру без явной необходимости
- Избегай лишних зависимостей
- Не редактируй secrets, env-файлы и конфигурации деплоя без явного запроса

## Git Workflow
Для любой кодовой задачи:
1. Сначала анализ
2. Потом план
3. Потом изменения
4. Потом локальная проверка
5. Потом commit
6. Потом push
7. Потом PR

## Required Checks Before Commit
- npm run build
- Проверка что нет случайных изменений вне области задачи
- Проверка что не изменились node_modules, dist и прочие артефакты

## UX/CRO Rules
- Каждый важный CTA должен быть понятен с первого взгляда
- Калькулятор должен не только считать, но и помогать забирать лид
- Любой новый текст должен быть простым, конкретным и коммерчески понятным
- Избегать абстрактных заголовков без выгоды
- Сначала предлагать минимальные изменения с максимальным эффектом

## Acceptance Criteria
Любая задача считается выполненной только если:
- проект собирается без ошибок
- UI не ломается на desktop и mobile
- не сломан текущий путь до лида
- изменения локализованы и понятны
- подготовлен понятный commit message
- подготовлен PR summary
